# Security policy

This is a small embedded game project for the MakeCode Arcade. If you discover a security issue — for example a way to brick the hardware via the firmware, or a malicious payload in a dependency — please report it privately first.

## Reporting

Email: **marcel.duetscher@gmail.com**

Please **don't** open a public issue for security problems — give us a chance to fix it first.

## What we treat as a security issue

- Firmware that can damage or brick the hardware
- Code that exfiltrates user data
- Build/CI compromise vectors (poisoned dependencies, malicious workflows, etc.)

## What we don't treat as a security issue

- Game-balance complaints
- Sprite or audio feedback (open a regular issue)
- "You can softlock the game by doing X" — that's a gameplay bug, file a normal issue
